## Key Decisions
- `ErrorWidget.builder` overridden globally in `main()` to render build-time exceptions on screen in release mode (instead of the default blank/black screen)
- `YouTubeService._yt` (`YoutubeExplode`) made lazy because its constructor may throw on iOS (network/HTTP client init); deferred to first actual method call
- `AppTheme.background` replaced with hardcoded `Color(0xFF121212)` in `main()` because `AppTheme.background` is a `Color` getter, not a const; using it inside `SystemUiOverlayStyle` constructor caused a release-mode error
- `youtube_explode_dart` bumped to `^2.3.4` for latest 2.x bugfixes
- CarPlay entitlement not pursued – requires Apple Developer Program approval; CarPlay scene config removed from Info.plist; all CarPlay Swift code removed after `import CarPlay` caused SIGKILL at launch
- `AudioService.init` wrapped with 8s timeout + fallback to plain `AudioPlayerHandler()` if it hangs

## Critical Context
- **Flutter 3.44.2 iOS template uses `SceneDelegate.swift`** – generated by `flutter create . --platforms=ios`. Without `UIApplicationSceneManifest` in Info.plist pointing to `SceneDelegate`, the Flutter engine never receives a window and renders nothing (black screen)
- **CarPlay is a restricted framework** – importing `CarPlay` in any Swift file and linking it without `com.apple.developer.carplay-audio` entitlement causes iOS to SIGKILL the process at launch
- **`AudioService.init` can hang on some iOS versions** due to AVAudioSession activation – 8s timeout fallback added
- **`youtube_explode_dart` v2.3.4** – `YoutubeExplode()` constructor may fail on iOS (HTTP client/platform channel); now lazily initialized
- **`ErrorWidget.builder`** override ensures widget build errors are visible on screen in release mode
- Spotify IPA not accessible from server – user has it at `C:\Users\safak\Downloads` on their local Windows machine
